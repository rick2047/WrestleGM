## Context
We are building a terminal-first, deterministic wrestling management MVP with Textual. The game is show-driven with a fixed 3-match card, data-driven roster and match types, and explicit simulation steps.

## Goals / Non-Goals
- Goals: deterministic simulation, keyboard-only Textual UI, data-driven content, show-by-show progression.
- Non-Goals: injuries, injuries UI, persistence to disk, non-textual UI, advanced booking rules.

## Decisions
- Decision: Keep state in-memory for the MVP and drive all randomness from a single seeded RNG.
- Decision: Use JSON files for wrestler and match type definitions to keep content data-driven.
- Decision: Implement explicit show lifecycle states and enforce booking validation before simulation.

## Risks / Trade-offs
- No persistence means sessions reset on app restart; acceptable for MVP.
- Using a minimal roster may limit variety, but ensures faster iteration.

## Migration Plan
- None for MVP; no existing data to migrate.

## Open Questions
- None for MVP.
